## Documented Divergence (Out-of-Range)

For `numUnacked > 2^31-1` (outside valid QUIC domain):

```
pn=2147483647, la=0 → numUnacked=2147483648
  Rust returns 5 (min_bits=33, div_ceil(8)=5)
  Lean model returns 4 (threshold model capped at 4)
```

**This is expected and correct** — the Lean model only guarantees
correspondence for valid QUIC inputs (`numUnacked ≤ 2^31-1`). This boundary
is explicitly documented in `FVSquad/PacketNumLen.lean` (see the
`pktNumLen_four_coverage` theorem, which requires `hquic` hypothesis).
The Rust implementation can return values > 4 for invalid inputs, which
a correct QUIC implementation should never produce.
