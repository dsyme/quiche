## What Is Tested

| Target | T45 — QPACK integer codec (RFC 7541 §5.1) |
|--------|-------------------------------------------|
| Rust source (encode) | `quiche/src/h3/qpack/encoder.rs:120` (`encode_int`) |
| Rust source (decode) | `quiche/src/h3/qpack/decoder.rs:212` (`decode_int`) |
| Lean model | `formal-verification/lean/FVSquad/QPACKInteger.lean` |

Test cases cover:

- **RFC 7541 §5.1 canonical examples** (cases 1–3): encode 10/1337/42 with
  various prefix widths; exactly matching the RFC specification and the
  Rust unit tests `encode_int1`, `encode_int2`, `encode_int3`.
- **Decode canonical examples** (cases 4–6): decode the same byte sequences;
  matching Rust `decode_int1`/`decode_int2`/`decode_int3`.
- **Single-byte boundary cases** (cases 7–13): zero values, values equal to
  `mask - 1` (last single-byte), values equal to `mask` (first multi-byte),
  prefix widths 1, 5, 7, 8.
- **Multi-byte cases** (cases 14–19): values > `mask` for various prefix
  widths, including 2-byte and 3-byte encodings.
- **Round-trip cases** (cases 20–25): `decodeInt (encodeInt v p) p = some v`
  for representative `(v, p)` pairs.

## Expected values

All expected byte sequences were derived by applying the RFC 7541 §5.1
algorithm manually and then cross-checked against the Rust unit tests in
`quiche/src/h3/qpack/encoder.rs` and `quiche/src/h3/qpack/decoder.rs`.

The Lean model uses `first = 0` (no flag bits in the high byte);
the expected values assume the same.
