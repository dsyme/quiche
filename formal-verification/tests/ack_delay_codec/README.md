## What Is Tested

| Target | T66 — ACK Delay Encode/Decode Codec |
|--------|--------------------------------------|
| Rust source | `quiche/src/lib.rs:4490` (encode), `lib.rs:8178` (decode) |
| Lean model | `formal-verification/lean/FVSquad/AckDelayCodec.lean` (`encode`/`decode`) |
| Valid domain | `exp ∈ {0..20}`, `delayMicros` any `u64` |

The Rust implementation uses:
- **Encode**: `ack_delay.as_micros() as u64 / 2u64.pow(ack_delay_exponent)`
- **Decode**: `ack_delay.checked_mul(2u64.pow(ack_delay_exponent))`

The Lean model uses:
- **Encode**: `delayMicros / 2^exp` (unbounded `Nat`)
- **Decode**: `encoded * 2^exp` (unbounded `Nat`)

The two are arithmetically equivalent when no `u64` overflow occurs. The
test cases exclude overflow cases (largest decode is `953674 * 2^20 ≈ 10^12`,
well within u64 range).

as expected.
