## Purpose

BBR2's ProbeBW state is divided into five *cycle phases*:
`NotStarted`, `Up`, `Down`, `Cruise`, `Refill`. Each phase operates with a
specific **pacing gain** and **cwnd gain** that control how aggressively the
sender probes for bandwidth or backs off.

The two methods `CyclePhase::pacing_gain` and `CyclePhase::cwnd_gain` perform
this dispatch, mapping the current phase to the appropriate gain from the BBR2
parameter set.

---

## Types

**`CyclePhase`** — five-variant enum:
```
NotStarted  Up  Down  Cruise  Refill
```

**Pacing gain** (`Params.probe_bw_*_pacing_gain`) — `f32`; modelled as an
exact fraction `num / den` (both `Nat`, `den > 0`).

**CWND gain** (`Params.probe_bw_*_cwnd_gain`) — `f32`; same rational model.

---

## Pacing Gain Assignment

`CyclePhase::pacing_gain(&self, params: &Params) -> f32`:

| Phase       | Returns                                  | Default value |
|-------------|------------------------------------------|---------------|
| `Up`        | `params.probe_bw_probe_up_pacing_gain`   | **1.25 = 5/4** |
| `Down`      | `params.probe_bw_probe_down_pacing_gain` | **0.90 = 9/10** |
| `NotStarted`| `params.probe_bw_default_pacing_gain`    | **1.00 = 1/1** |
| `Cruise`    | `params.probe_bw_default_pacing_gain`    | **1.00 = 1/1** |
| `Refill`    | `params.probe_bw_default_pacing_gain`    | **1.00 = 1/1** |

Phases `NotStarted`, `Cruise`, `Refill` all share a single gain — the
"default" pacing gain. Only `Up` uses the elevated gain; only `Down` uses
the below-default gain.

## CWND Gain Assignment

`CyclePhase::cwnd_gain(&self, params: &Params) -> f32`:

| Phase       | Returns                           | Default value |
|-------------|-----------------------------------|---------------|
| `Up`        | `params.probe_bw_up_cwnd_gain`    | **2.25 = 9/4** |
| `Down`      | `params.probe_bw_cwnd_gain`       | **2.00 = 2/1** |
| `NotStarted`| `params.probe_bw_cwnd_gain`       | **2.00 = 2/1** |
| `Cruise`    | `params.probe_bw_cwnd_gain`       | **2.00 = 2/1** |
| `Refill`    | `params.probe_bw_cwnd_gain`       | **2.00 = 2/1** |

All non-`Up` phases use the same `probe_bw_cwnd_gain`. Only `Up` uses the
elevated `probe_bw_up_cwnd_gain`.

## Preconditions

None for the gain lookup functions themselves (they are total).

For meaningful inequality theorems about the default parameter values, we
require the rational model denominators to be positive (trivially satisfied
by the concrete defaults 4, 10, 1, 4, 1).

---

## Postconditions / Key Properties

1. **`Up` pacing gain is super-unity** (`5 > 4`): the `Up` phase sends faster
   than the estimated bandwidth.

2. **`Down` pacing gain is sub-unity** (`9 < 10`): the `Down` phase sends
   slower to drain the queue.

3. **Default pacing gain is unity** (`1 = 1`): phases `NotStarted`, `Cruise`,
   `Refill` send at exactly the estimated bandwidth.

4. **Pacing gain ordering**: `Down < Default < Up` (9/10 < 1/1 < 5/4).

5. **`Up` cwnd gain is super-unity** (`9 > 4`): the `Up` phase allows a
   larger window.

6. **Non-`Up` cwnd gain is super-unity** (`2 > 1`): even the default cwnd
   gain is above 1.

7. **`Up` cwnd gain > non-`Up` cwnd gain** (`9/4 > 2/1`, i.e. `9 > 8`).

8. **Phase partition for pacing gain**: exactly `{Up}` uses elevated gain,
   exactly `{Down}` uses sub-default gain, exactly `{NotStarted, Cruise,
   Refill}` use the unity default.

9. **Phase partition for cwnd gain**: exactly `{Up}` uses the elevated cwnd
   gain; `{NotStarted, Down, Cruise, Refill}` use the same lower gain.

10. **`applyGain` monotone in bandwidth**: for fixed phase and denominator,
    a higher bandwidth yields a higher (or equal) adjusted bandwidth.

---

## Invariants

- All five phases return a *positive* gain (numerator > 0 for any
  non-trivial params).
- The gain assignment is *total*: every phase maps to exactly one of
  the three pacing-gain buckets and one of the two cwnd-gain buckets.

---

## Edge Cases

- `NotStarted` is the default value after `Cycle::default()`. Its pacing
  and cwnd gains are the same as `Cruise` and `Refill` — i.e. the "cruise"
  default.
- There is no phase that uses both an elevated pacing gain and a non-elevated
  cwnd gain, or vice versa in the pacing-gain direction. The Up phase uses
  both elevated gains simultaneously.
